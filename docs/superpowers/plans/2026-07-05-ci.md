# CI (Plan 5b) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a GitHub Actions CI workflow that gates every push to `main` and every PR on the migration guards, ESLint, a new typecheck, the full vitest suite, and the web build.

**Architecture:** One workflow (`.github/workflows/ci.yml`) with three jobs — `check` (fast, no services), then `test` and `build` (both `needs: check`). `check` also runs a new `typecheck` script that covers production source across the workspaces. `test` runs Meilisearch + MinIO as containers (Postgres is self-provisioned by ingest's Testcontainers); `build` runs `next build` hermetically with placeholder env.

**Tech Stack:** GitHub Actions, Node 20, npm workspaces, TypeScript (`tsc --noEmit`), vitest, Testcontainers, Docker.

## Global Constraints

- Monorepo is npm workspaces rooted at `app/` (`core`, `db`, `ingest`, `search`, `web`). Every CI job sets `working-directory: app`. `engines.node >= 20` → CI uses Node **20**.
- Test services & credentials (verbatim, matching what the tests default to): Meilisearch at `http://localhost:7700`, master key `masterKey`; MinIO (S3) at `http://localhost:9000`, access/secret `minioadmin`/`minioadmin`. Test env vars: `TEST_MEILI_HOST`, `TEST_MEILI_KEY`, `MEILI_HOST`, `TEST_S3_ENDPOINT`, `TEST_S3_ACCESS_KEY`, `TEST_S3_SECRET_KEY`.
- Postgres is **not** a CI service — `ingest` self-provisions it via `@testcontainers/postgresql` using the runner's Docker socket. Web action tests mock the db (`vi.mock`); `DATABASE_URL` is only read by web *production* code. The S3 tests create/delete their own unique buckets via `ensureBucket`/`nukeBucket`, so **no bucket-creation step is needed**.
- `typecheck` covers **production source**. `ingest` and `web` exclude their test files (they depend on runtime type augmentation standalone `tsc` can't see — Better Auth plugin inference in `ingest/test/auth.test.ts`; `@testing-library/jest-dom` matchers in `web/src/**/__tests__`). `core`, `db`, `search` typecheck fully (green today, tests included).
- The test command is the root aggregator `npm test` (→ `npm run test --workspaces --if-present`), NOT `npm test --workspaces` (which would fail on `db`, which has no `test` script).
- OUT of scope: e2e/Playwright, deploy/release, branch-protection settings, the Python data pipeline, Node-version matrix, caching beyond npm.
- Env quirk for **local** runs only (not CI): `~/.npm` is root-owned, so prefix local `npm`/`npx` with `NPM_CONFIG_CACHE=<scratchpad>/npm-cache` (session-specific scratchpad path). CI runners are unaffected.
- Conventional Commits.

### Task 1: `typecheck` scripts + src-only configs

**Files:**
- Create: `app/ingest/tsconfig.typecheck.json`, `app/web/tsconfig.typecheck.json`
- Modify: `app/package.json`, `app/core/package.json`, `app/db/package.json`, `app/ingest/package.json`, `app/search/package.json`, `app/web/package.json`

**Interfaces:**
- Produces: `npm run typecheck` (in `app/`) — runs `tsc --noEmit` across all five workspaces, exit 0 on the current tree. Consumed by the CI `check` job (Task 2).

- [ ] **Step 1: Add the two src-only tsconfigs**

`app/ingest/tsconfig.typecheck.json`:
```json
{ "extends": "./tsconfig.json", "include": ["src"] }
```

`app/web/tsconfig.typecheck.json`:
```json
{
  "extends": "./tsconfig.json",
  "exclude": ["node_modules", "**/__tests__/**", "**/*.test.ts", "**/*.test.tsx", "e2e"]
}
```

- [ ] **Step 2: Add per-workspace `typecheck` scripts**

`app/core/package.json`, `app/db/package.json`, `app/search/package.json` — add to `"scripts"`:
```json
    "typecheck": "tsc --noEmit"
```

`app/ingest/package.json` — add to `"scripts"`:
```json
    "typecheck": "tsc --noEmit -p tsconfig.typecheck.json"
```

`app/web/package.json` — add to `"scripts"` (keep existing `dev`/`build`/`start`/`lint`/`test`/`e2e`):
```json
    "typecheck": "tsc --noEmit -p tsconfig.typecheck.json"
```

- [ ] **Step 3: Add the root aggregator**

`app/package.json` — add to `"scripts"` (keep `test` and `db:generate`):
```json
    "typecheck": "npm run typecheck --workspaces --if-present"
```

- [ ] **Step 4: Verify it is green on the current tree**

Run (from `app/`, prefixing with the local `NPM_CONFIG_CACHE` per Global Constraints):
```bash
npm run typecheck
echo "exit=$?"
```
Expected: each workspace's `tsc --noEmit` runs; overall `exit=0`. If a workspace fails, confirm the failures are in *test* files only — if so the src-only config for that workspace is wrong; if a *src* file fails, that is a real pre-existing type error: STOP and report it (do not silence it).

```bash
git add app/package.json app/core/package.json app/db/package.json app/ingest/package.json app/search/package.json app/web/package.json app/ingest/tsconfig.typecheck.json app/web/tsconfig.typecheck.json

### Task 2: `.github/workflows/ci.yml`

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: `npm run check`/`npm run verify` (`@revelio/db`, Plan 5a), `npm run lint -w web`, `npm run typecheck` (Task 1), `npm test`, `npm run build -w web`.

- [ ] **Step 1: Local pre-flight for the `build` job env**

`next build` must succeed without live services. From `app/`, run it with a minimal placeholder env to discover the required vars:
```bash
DATABASE_URL='postgres://user:pass@localhost:5432/db' \
MEILI_HOST='http://localhost:7700' MEILI_SEARCH_KEY='masterKey' \
BETTER_AUTH_SECRET='ci-placeholder-secret' BETTER_AUTH_URL='http://localhost:3000' \
NEXT_PUBLIC_BASE_URL='http://localhost:3000' NEXT_PUBLIC_IMAGE_BASE_URL='http://localhost:9000/card-images' \
ADMIN_EMAILS='admin@example.com' \
npm run build -w web
echo "exit=$?"
```
Expected: `next build` completes, `exit=0`. If it fails complaining about a missing env var, add that var (placeholder value) to both this command and the `build` job's `env:` in Step 2. If it needs a live DB/Meili at build time (it should not — pages render dynamically), STOP and report.

- [ ] **Step 2: Write the workflow**

`.github/workflows/ci.yml`:
```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

defaults:
  run:
    working-directory: app

jobs:
  check:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: app
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
          cache-dependency-path: app/package-lock.json
      - run: npm ci
      - run: npm run check -w @revelio/db
      - run: npm run verify -w @revelio/db
      - run: npm run lint -w web
      - run: npm run typecheck

  test:
    needs: check
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: app
    env:
      TEST_MEILI_HOST: http://localhost:7700
      TEST_MEILI_KEY: masterKey
      MEILI_HOST: http://localhost:7700
      TEST_S3_ENDPOINT: http://localhost:9000
      TEST_S3_ACCESS_KEY: minioadmin
      TEST_S3_SECRET_KEY: minioadmin
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
          cache-dependency-path: app/package-lock.json
      - name: Start Meilisearch
        run: docker run -d --name meili -p 7700:7700 -e MEILI_MASTER_KEY=masterKey -e MEILI_NO_ANALYTICS=true getmeili/meilisearch:v1.10
      - name: Start MinIO
        run: docker run -d --name minio -p 9000:9000 -e MINIO_ROOT_USER=minioadmin -e MINIO_ROOT_PASSWORD=minioadmin minio/minio server /data
      - name: Wait for services
        run: |
          for i in $(seq 1 30); do curl -fsS http://localhost:7700/health && break || sleep 2; done
          for i in $(seq 1 30); do curl -fsS http://localhost:9000/minio/health/live && break || sleep 2; done
      - run: npm ci
      - run: npm test

  build:
    needs: check
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: app
    env:
      DATABASE_URL: postgres://user:pass@localhost:5432/db
      MEILI_HOST: http://localhost:7700
      MEILI_SEARCH_KEY: masterKey
      BETTER_AUTH_SECRET: ci-placeholder-secret
      BETTER_AUTH_URL: http://localhost:3000
      NEXT_PUBLIC_BASE_URL: http://localhost:3000
      NEXT_PUBLIC_IMAGE_BASE_URL: http://localhost:9000/card-images
      ADMIN_EMAILS: admin@example.com
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
          cache-dependency-path: app/package-lock.json
      - run: npm ci
      - run: npm run build -w web
```

> If Step 1 added any env var, mirror it into the `build` job's `env:` here.

- [ ] **Step 3: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions workflow (migrations/lint/typecheck/test/build)"
```

- [ ] **Step 4: Push and verify the CI run**

```bash
git push -u origin <branch>
```
Then inspect the run (`gh run watch` / `gh run view --log`, or the Actions tab). Expected: all three jobs (`check`, `test`, `build`) pass green.

- If `check` fails: re-run the failing command locally (Task 1 Step 4 / the specific `-w` command) and fix.
- If `test` fails on connection refused: the Meilisearch/MinIO wait loop was too short or the health path differs — extend retries or fix the URL. If a test needs a service not started here (e.g. an unexpected Postgres dependency), add it as a `docker run` step mirroring the others.
- If `build` fails on a missing env var: add it (placeholder) to the `build` job `env:` and re-push.

Iterate (amend/commit + push) until the run is green before finishing.
