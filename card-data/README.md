# revelio.cards – Data Model

Derived from [Tressley/hpjson](https://github.com/Tressley/hpjson) (the data behind
accio.cards) and transformed into a clean, consistent, multilingual schema.

## Layout

```
card-data/
  card.schema.json          # JSON Schema (Draft-07) of a card (multilingual)
  sets.schema.json          # JSON Schema of set metadata
  transform_hpjson.py       # hpjson card -> revelio card (English)
  build_dataset.py          # single entry point: transform + merge + sets + index
  accio_images.py           # download card images + set symbols from accio
  image_overrides.json      # id -> correct accio filename (typo/name fixes)
  translations/<lang>.json  # editable per-language text overlays (id -> fields)
  i18n/labels.<lang>.json   # UI label dictionaries (types, lessons, rarities)
  DATABASE-CHOICE.md        # which DB fits (PostgreSQL)
  IMAGES-SOURCING.md        # where card images & symbols come from
  STRUCTURE-REVIEW.md       # structure notes / rationale
  dist/                     # GENERATED (git-ignored) – build output
    cards.json              #   full bundle, all languages
    sets.json               #   set metadata keyed by setCode
    cards.<lang>.json       #   slim, denormalized per language (with fallback)
    search-index.<lang>.json#   folded name/text/flavor for fast search
  assets/                   # GENERATED (git-ignored) – downloaded images
    cards/<id>.png          #   card image (+ thumb/<id>.jpg)
    symbols/<code>.png      #   set symbols (all PNG)
```

Editable sources are small and git-friendly; **`dist/` is a build artifact** (do not
hand-edit, do not commit).

## Build

```bash
python3 build_dataset.py            # clones hpjson if missing, builds -> dist/, removes the clone
```
hpjson (the English source) is cloned automatically into `card-data/hpjson/` if it isn't
there, and the clone is **removed again after the build** — so a default run leaves only
`dist/` behind. Options:

- `python3 build_dataset.py --keep-hpjson` — keep the auto-clone for reuse.
- `python3 build_dataset.py path/to/hpjson` — use an existing checkout; a checkout you
  pass in is **never** deleted (only a clone the script created itself is).

The build transforms hpjson, merges all `translations/<lang>.json` overlays, derives
`sets.json`, writes per-language bundles + search indexes, and validates everything
against the schemas (0 errors expected).

## Dataset at a glance

- **1,035 cards** across **14 sets**.
- **504** official WotC cards (2001–2003), **531** fan/Revival cards. Each set's
  `isOfficial` flag lives in `sets.json` → to get "the 2001 game" filter cards whose
  `setCode` is official (BS, QC, DA, AAH, COS).
- Card types: Spell, Item, Creature, Character, Adventure, Location, Event,
  Lesson, Match (+ a few multi-type cards).
- Rarities: Common, Uncommon, Rare, Lesson — plus a separate `finish`
  (normal / foil / holo). Premium printings = base rarity + foil/holo finish, e.g.
  Draco Malfoy is `rarity: Rare` with `finish: holo` (or `foil`). The base rarity of
  premium-only cards isn't in the source and defaults to `Rare`.

## Schema shape

- **Card** (`card.schema.json`): language-independent gameplay at top level — `id`,
  `setCode` (→ sets.json), `number`, `types[]`, `subTypes[]`, `lesson`, `cost`,
  `provides[]`, `rarity`, `finish` (normal/foil/holo), `artist[]`,
  `stats{health,damagePerTurn}`, `orientation`,
  `legality`, `draftValue`, `rulings[]` — plus `name` (canonical EN, derived),
  `defaultLanguage`, `languages[]`, and `localizations`.
- **localizations.<lang>**: the printed face per language — `name`, `status`,
  `source`, `text`, `flavorText`, `adventure{effect,toSolve,reward}` /
  `match{toWin,prize}`, `image{file,url}`. (For Adventure/Match cards `text` is null;
  the structured block is canonical.)
- **Set** (`sets.schema.json`): `code`, `name`, `releaseDate`, `isOfficial`,
  `cardCount`, `symbol`.

Type/lesson/rarity keys stay canonical English for filtering; their **display labels
are translated centrally** via `i18n/labels.<lang>.json`, not per card.

`name` and `languages[]` are **derived** (regenerated by the build) — never hand-edit.

## Translation provenance (hybrid strategy)

There is **no ready-made dataset** of the official translated card text (e.g. German
AMIGO wording). Each localization carries a `status`:

- `official` – real printed text (English comes from hpjson).
- `machine` – machine translation (full coverage, **not** the official wording).
- `community` – contributed by the community.
- `unknown` – provenance unclear.

Plan: ship machine translations now (clearly flagged), replace with
`official`/`community` over time. The UI shows a badge and falls back to
`defaultLanguage` when a translation is missing.

Currently **`en`** = official (all 1,035 cards), **`de`** = machine (Base Set, 118).

## Adding a language

1. Create `translations/<code>.json` mapping card `id` → `{name, text, flavorText,
   status, source}` (optionally `adventure`/`match`/`image`).
2. Add `i18n/labels.<code>.json` for the UI labels (see `labels.de.json`).
3. Re-run `python3 build_dataset.py hpjson`.

## Images & symbols

Card images and set symbols are downloaded from accio / the Revival site via
`accio_images.py` (run locally) into `assets/` — see `IMAGES-SOURCING.md` for the full
flow and the rights note. Quick version:

```bash
python3 -m pip install pillow      # optional, for thumbnails
python3 accio_images.py --download # cards -> assets/cards/<id>.png, symbols -> assets/symbols/<code>.png
```

- Every file is named by the card **id** (or set **code** for symbols).
- `image_overrides.json` maps ids to the correct accio filename where the source data
  has typos/name mismatches.
- `--link` mode instead sets `image.url` to the remote accio URL (no download).
- Coverage: 1,035/1,035 card images, 13/13 set symbols (all PNG).
