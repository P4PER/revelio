# CI (Plan 5b) — Design

## Goal

Add continuous integration (GitHub Actions) that runs on every push to `main` and every pull request, gating merges on: the migration guards (Plan 5a), lint, typecheck, the full vitest suite, and the web build. No CI exists today; the repo has a GitHub remote (`github.com/P4PER/revelio.git`).

Explicitly **out of scope** (deferred): Playwright e2e (removed to avoid the gitignored seed-data problem — `card-data/dist` and `card-data/assets` are generated locally by Python scripts from external sources, so a fresh CI checkout has no data); deploy/release automation; branch-protection settings; the data-build pipeline.

## Verified facts (current repo state)

- Monorepo: npm workspaces rooted at `app/` (`core`, `db`, `ingest`, `search`, `web`); `engines.node >= 20`.
- `app` scripts: `test` → `npm run test --workspaces --if-present`; `db:generate`.
- Per-workspace scripts: `core`/`ingest`/`search` → `vitest run`; `web` → `dev`/`build` (`next build`)/`start`/`lint` (`eslint`)/`test` (`vitest run`)/`e2e`; `db` → `generate`/`migrate`/`check` (`drizzle-kit check`)/`verify` (`node scripts/verify-migrations.mjs`, from Plan 5a).
- **No `typecheck` script exists** in any workspace. vitest runs via tsx (transpile-only, no type errors surfaced); only `next build` typechecks `web`.
- Test service topology is **mixed**:
  - `ingest` self-provisions Postgres via `@testcontainers/postgresql` (`PostgreSqlContainer`) — needs a Docker socket (present on GitHub ubuntu runners).
  - `search` tests connect to an **external** Meilisearch: `TEST_MEILI_HOST` (default `http://localhost:7700`), `TEST_MEILI_KEY` (default `masterKey`).
  - `web` tests connect to external Meilisearch (`TEST_MEILI_HOST`/`MEILI_HOST`) and, for image tests, MinIO/S3 (`TEST_S3_ENDPOINT` default `http://localhost:9000`, `TEST_S3_ACCESS_KEY`/`TEST_S3_SECRET_KEY` default `minioadmin`).
  - `core` tests are pure units (no services).

## Architecture

One workflow file `.github/workflows/ci.yml`. Triggers: `push` to `main`, `pull_request`. Ubuntu-latest, `actions/setup-node@v4` with Node 20 and npm cache keyed on `app/package-lock.json`. All jobs use `defaults.run.working-directory: app`.

Three jobs; `test` and `build` declare `needs: check` so the cheap gate blocks the expensive jobs (fail-fast, clear signal). Rejected alternatives: a single sequential job (no parallelism, noisier failures) and a Node-version matrix (the app targets one runtime).

### Job 1 — `check` (fast gate, no Docker, no services)

Steps (after checkout + setup-node + `npm ci`):
1. `npm run check -w @revelio/db` (drizzle-kit journal/snapshot consistency)
2. `npm run verify -w @revelio/db` (Plan 5a drift guard — schema vs. migrations; no DB)
3. `npm run lint -w web` (ESLint)
4. `npm run typecheck` (**new** — see below)

### Job 2 — `test` (needs: check; Docker + services)

- Service containers on the runner:
  - `getmeili/meilisearch:v1.10` published on `7700`, env `MEILI_MASTER_KEY=masterKey`, `MEILI_NO_ANALYTICS=true` (mirrors compose).
  - `minio/minio` published on `9000` with `minioadmin`/`minioadmin` (server command `server /data`), plus a step to create the `card-images` bucket if any test needs it.
- Job env pointing tests at those services: `TEST_MEILI_HOST=http://localhost:7700`, `TEST_MEILI_KEY=masterKey`, `TEST_S3_ENDPOINT=http://localhost:9000`, `TEST_S3_ACCESS_KEY=minioadmin`, `TEST_S3_SECRET_KEY=minioadmin`.
- Postgres is **not** a service — `ingest` self-provisions it via Testcontainers using the runner's Docker socket.
- Steps: `npm ci` → `npm test --workspaces` (vitest across core, ingest, search, web).

> Implementation note: the exact service-container syntax (image, `ports`, `options` for healthchecks) and the precise env-var set are finalized during implementation by running the suite locally against the same containers and matching whatever the tests actually read. MinIO healthcheck/bucket-creation is only wired if a test fails without it.

### Job 3 — `build` (needs: check)

- Steps: `npm ci` → `npm run build -w web` (`next build`).
- Provide placeholder build-time env for the vars the build reads (e.g. `NEXT_PUBLIC_BASE_URL`, `NEXT_PUBLIC_IMAGE_BASE_URL`, and any that must be present at build) so the build is hermetic — no live services. The exact set is confirmed during implementation by running `next build` locally with a minimal env.

## New code (beyond the workflow file)

Add a typecheck script so CI catches type errors in the non-web packages (today only `web` is typechecked, and only via `next build`):

- Per workspace (`core`, `db`, `ingest`, `search`, `web`): `"typecheck": "tsc --noEmit"` in `package.json` scripts. Each workspace already has (or gets) a `tsconfig.json` extending `tsconfig.base.json`; `web` uses its Next.js `tsconfig.json`.
- Root `app/package.json`: `"typecheck": "npm run typecheck --workspaces --if-present"`.

Implementation verifies `npm run typecheck` is green locally before wiring it into CI; any pre-existing type errors it surfaces are fixed as part of the task (or, if out of proportion, flagged back to the user).

## Testing / verification

A workflow can't be fully exercised offline, so verification is two-layer:

1. **Local pre-flight** (before pushing): run each job's commands locally and confirm green —
   - `check`: `npm run check -w @revelio/db`, `npm run verify -w @revelio/db`, `npm run lint -w web`, `npm run typecheck`.
   - `test`: start local Meilisearch + MinIO containers (compose or `docker run`), then `npm test --workspaces`.
   - `build`: `npm run build -w web` with the placeholder env.
2. **CI run**: push the branch, open/inspect the Actions run, confirm all three jobs pass. Iterate on service/env wiring against the real runner if anything differs from local.

## Scope / decomposition

Small and cohesive — one plan:
1. Add `typecheck` scripts (per-workspace + root aggregator); make `npm run typecheck` green locally.
2. Write `.github/workflows/ci.yml` (three jobs) and validate via local pre-flight + a real CI run on the branch.

- **OUT:** e2e/Playwright; deploy/release; branch protection; the Python data pipeline; caching beyond npm; multi-Node matrix.
