# GHCR Image Publish Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `web` and `ingest` Docker images in GitHub Actions and publish them to GitHub Container Registry (GHCR) as private packages on every merge to `main` and on manual dispatch.

**Architecture:** A new `publish` job is added to the existing `.github/workflows/ci.yml`, gated on the `check`/`test`/`build` jobs. It builds two images from the `app/` workspace root: a Next.js **standalone** web image (with `NEXT_PUBLIC_*` baked from GitHub secrets) and a code-only **ingest** image (dataset mounted at runtime). Both push to `ghcr.io/p4per/revelio-{web,ingest}` tagged `latest` + `sha-<short>`.

**Tech Stack:** GitHub Actions, `docker/build-push-action` + `buildx` (gha cache), Node 22, npm workspaces, Next.js 16 standalone output, `tsx`.

## Global Constraints

- **Node version:** `>=22` everywhere (CI, Dockerfiles, `engines`). Copy exact base image `node:22-alpine`.
- **Workspace root:** all Docker build contexts are `app/` (where `package-lock.json` lives). Dockerfiles `COPY` paths are relative to `app/`.
- **Image names (lowercase, private):** `ghcr.io/p4per/revelio-web`, `ghcr.io/p4per/revelio-ingest`. `docker/metadata-action` lowercases the owner automatically.
- **Tags per image:** `latest` and `sha-<short-commit>`.
- **Publish triggers only:** `push` to `main` or `workflow_dispatch`. Never on pull requests.
- **Baked into web image at build (from GitHub secrets):** `NEXT_PUBLIC_BASE_URL`, `NEXT_PUBLIC_IMAGE_BASE_URL`. **Never baked:** `DATABASE_URL`, `MEILI_*`, `BETTER_AUTH_*`, `S3_*`, `ADMIN_EMAILS` (host supplies at runtime).
- **Ingest is code-only:** `DATA_DIR`/`I18N_DIR`/`ASSETS_DIR` are runtime mounts, never baked.
- **Commits:** Conventional Commits.

**Note on verification style:** This is infrastructure work (Dockerfiles + YAML); there are no unit tests to write. Each task's "test" is a build/run command with an expected result. Run them exactly as written.

```dockerfile

WORKDIR /app
COPY package.json package-lock.json ./
COPY core/package.json ./core/package.json
COPY db/package.json ./db/package.json
COPY search/package.json ./search/package.json
COPY ingest/package.json ./ingest/package.json

# --- build: compile the Next.js app in standalone mode ---
FROM node:22-alpine AS build
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY tsconfig.base.json ./
COPY core ./core
COPY db ./db
COPY search ./search
COPY web ./web

# NEXT_PUBLIC_* are inlined into the client bundle at build time.
ARG NEXT_PUBLIC_BASE_URL
ARG NEXT_PUBLIC_IMAGE_BASE_URL
ENV NEXT_PUBLIC_BASE_URL=$NEXT_PUBLIC_BASE_URL
ENV NEXT_PUBLIC_IMAGE_BASE_URL=$NEXT_PUBLIC_IMAGE_BASE_URL

# Placeholder server env so `next build` can evaluate server modules
# (mirrors the CI build job). NOT inlined into the client bundle.
ENV DATABASE_URL=postgres://user:pass@localhost:5432/db \
    MEILI_HOST=http://localhost:7700 \
    MEILI_SEARCH_KEY=build-placeholder \
    BETTER_AUTH_SECRET=build-placeholder-secret \
    BETTER_AUTH_URL=http://localhost:3000 \
    ADMIN_EMAILS=admin@example.com

RUN npm run build -w web

# --- runtime: minimal standalone server ---
FROM node:22-alpine AS runtime
WORKDIR /app
ENV NODE_ENV=production
ENV PORT=3000 HOSTNAME=0.0.0.0
RUN addgroup -g 1001 -S nodejs && adduser -S nextjs -u 1001

# Standalone output preserves the monorepo layout: server.js lives under web/.
COPY --from=build --chown=nextjs:nodejs /app/web/.next/standalone ./
COPY --from=build --chown=nextjs:nodejs /app/web/.next/static ./web/.next/static
COPY --from=build --chown=nextjs:nodejs /app/web/public ./web/public

USER nextjs
EXPOSE 3000
CMD ["node", "web/server.js"]
```

### Task 4: Add the GHCR publish job to CI

**Files:**
- Modify: `.github/workflows/ci.yml` (append `publish` job)

**Interfaces:**
- Consumes: the `check`/`test`/`build` jobs (gate); the web Dockerfile (Task 2) and ingest Dockerfile (Task 3).
- Produces: `ghcr.io/p4per/revelio-web` and `ghcr.io/p4per/revelio-ingest`, tagged `latest` + `sha-<short>`.

- [ ] **Step 1: Append the publish job**

At the end of `.github/workflows/ci.yml`, after the `build` job, add:

```yaml
  publish:
    needs: [check, test, build]
    if: >-
      github.event_name == 'workflow_dispatch' ||
      (github.event_name == 'push' && github.ref == 'refs/heads/main')
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    steps:
      - uses: actions/checkout@v4

      - uses: docker/setup-buildx-action@v3

      - uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - id: meta-web
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/${{ github.repository_owner }}/revelio-web
          tags: |
            type=raw,value=latest
            type=sha,prefix=sha-,format=short

      - uses: docker/build-push-action@v6
        with:
          context: app
          file: app/web/Dockerfile
          push: true
          tags: ${{ steps.meta-web.outputs.tags }}
          labels: ${{ steps.meta-web.outputs.labels }}
          build-args: |
            NEXT_PUBLIC_BASE_URL=${{ secrets.NEXT_PUBLIC_BASE_URL }}
            NEXT_PUBLIC_IMAGE_BASE_URL=${{ secrets.NEXT_PUBLIC_IMAGE_BASE_URL }}
          cache-from: type=gha,scope=web
          cache-to: type=gha,mode=max,scope=web

      - id: meta-ingest
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/${{ github.repository_owner }}/revelio-ingest
          tags: |
            type=raw,value=latest
            type=sha,prefix=sha-,format=short

      - uses: docker/build-push-action@v6
        with:
          context: app
          file: app/ingest/Dockerfile
          push: true
          tags: ${{ steps.meta-ingest.outputs.tags }}
          labels: ${{ steps.meta-ingest.outputs.labels }}
          cache-from: type=gha,scope=ingest
          cache-to: type=gha,mode=max,scope=ingest
```

Note: the workflow's top-level `defaults.run.working-directory: app` only affects `run:` steps; the `uses:` action steps here resolve `context: app` relative to the repo root, which is correct.
